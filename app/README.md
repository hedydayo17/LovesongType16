# ラブソング診断16 — 動くプロトタイプ

Phase 3〜6 を通しで実装した、ブラウザだけで動く静的サイト(ビルド不要・モバイル最優先)。

## 動かし方

このフォルダ(`app/`)で簡易サーバを立てて開くだけ。

```bash
cd app
python3 -m http.server 8765
# ブラウザで http://localhost:8765/index.html を開く
```

`file://` 直開きでも動くが、`localStorage`(👍👎フィードバック)挙動の確認はサーバ経由推奨。

## フロー

```
ヒーロー(landing) → 生年月日(必須・カード非表示) → ジャンル(0〜5・Skip可) → アーティスト(0〜3・Skip可)
 → 20問(16P式スケール・自動送り) → 【スクロール結果(多セクション)】→ 相性シェア
ヒーロー / 結果 → 16タイプ図鑑 → 個別タイプ詳細
```

ラブタイプ診断64 / 16personalities に寄せた設計:
- **見た目**:ラブタイプ64寄りの明るいポップ系(白〜クリーム基調・ピンク/ゴールド・大きな丸み・ソフトな影・タイプ色を主役に)
- **ヒーロー**:信頼バッジ・「3分/無料/登録不要」・大型CTA・図鑑導線
- **回答**:**16personalities 式の「そう思う⇔思わない」7段階スケール × 20ステートメント**(`questions.js`)。1タップ自動送り・スライド遷移・戻る復元・「あと○問」
- **結果**:タイプ表示(タイプ色の全面カード)→ タグライン → 解説 → 恋愛の強み+愛されポイント → こんな恋+あるある → 相性◎/△ → "10"曲 → シェアカード
- **拡散**:「気になる人に送って相性チェック」+ #ラブソング診断16
- **図鑑**:16タイプのカラーカード一覧 + 個別読み物 → 診断へ誘導

### 回答ロジック
`questions.js` の各ステートメントは `w`(タイプ別重み)を持つ。回答位置 0..6 を `SCALE_VALUES`(+3..-3)に変換し `w` を掛けて加点。最高点のタイプが結果(同点は `TIE_PRIORITY`)。全16タイプ到達可能を確認済み。

### 結果体験(Spotify Wrapped風)
診断後はフルビューポートのパネルが scroll-snap で1画面ずつ吸い付き、各パネルが見えると要素が順次せり上がる(`IntersectionObserver`)。
パネル構成:導入 → タイプ表示(全面カラー) → タグライン → 解説 → 相性 → "10"カウント → 曲リスト(再シャッフル / 👍👎) → スクショ用シェアカード。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | エントリ。4つのJSを読むだけ |
| `css/style.css` | モバイル最優先・縦画面・スクショ映えスタイル |
| `js/data.js` | 16タイプ定義＋詳細解説＋カラー/モチーフ＋相性＋同点優先順位。**正本** |
| `js/questions.js` | 15問＋配点表(questions.md と一致) |
| `js/songs.js` | **モック楽曲プール(Phase 4スタブ)**。`IS_MOCK` フラグあり |
| `js/mascot.js` | 16タイプの**手続き的SVGキャラ生成**(`mascotSVG`)。タイプ色の体+顔+タイプ別トッパー |
| `js/motion.js` | **モーションエンジン(`MX`)**。Lenis慣性スクロール・背景FX(メッシュ/コニック/ブロブ/粒子)・ワイプ遷移・キネティック入場・スケール弾け・結果のScrollTrigger(reveal/char/%バー/カウント/マーキー/紙吹雪)。GSAP無し/reduce-motionでも内容は表示 |
| `js/app.js` | 画面遷移・スコア集計・タイプ判定・レコメンド・%バー・シェア・フィードバック |
| `js/vendor/` | gsap / ScrollTrigger / lenis(**セルフホスト**) |

## 依存(セルフホスト)
- GSAP 3.13 + ScrollTrigger + Lenis 1.1.14 を `js/vendor/` に同梱(CDN改ざんリスク回避・オフライン動作可)。
- 未読込/reduce-motion でもフォールバックで内容は表示・操作可能。
- 開発中にCSS/JSが古いまま表示される場合はハードリロード(リンクに `?v=` 付与済み)。

## 本番化で差し替えるところ(現状スタブ)

1. **楽曲ソース**:`js/songs.js` のモック配列 → Spotify Web API + LLM タグ付け結果。`IS_MOCK = false` にすると「サンプル楽曲」表記が消える。
2. **レコメンド**:`app.js` の `recommend()` は「タイプ一致+5 / 相性+2 / ジャンル+2」の重み付き抽選。本番もこの重みロジックは流用可。Spotifyのプレビュー再生・ジャケット画像・外部リンクを `songRow()` に追加する。
3. **フィードバック収集**:現在 `localStorage`(`ongaku_fb`)。本番はサーバ/DBに送信し、3〜6ヶ月後の配点・スコア再学習に使う。
4. **生年月日**:現在は世代マッチ未使用(将来用に保持)。Phase 4で年代フィルター/重みを `recommend()` に追加。
