# 運用ルール / デザイン原則

## 1. ポジティブ命名原則(最優先)

**全16タイプ、どれが結果として出ても "シェアして自慢/可愛がりたくなる" フレーミングにする**。

### 絶対ルール
- メンヘラ / 重め / 拗らせ / 軽率 / 量産型 などの**自虐スラング**をそのままラベルにしない
- 「中身」(タイプの行動・特性記述)には全スペクトラム(メンヘラ的・軽率的・拗らせ的)を含めてよい/含めるべき
- 「ラベル」は必ずポジティブな再フレーム

### 判定基準(これがYESになるラベルだけ採用)
1. 友達にこのタイプって言われたら嬉しいか?
2. 自分のプロフィールに貼りたいか?
3. TikTokでスクショしてシェアしたくなるか?

### Why
2026-05-19のやりとりで、当初「メンヘラ系」のような温度感の命名を提案 → ユーザーから「メンヘラって出たらシェアしたくないよね」と指摘。自虐は本人が自発的に使う時だけ成立する語で、診断結果として外部から貼られると侮辱感になり拡散しない。MBTIが「うるさい外向人間」じゃなく「運動家」、LoveType16が「重い女」じゃなく「忠犬ハチ公」と命名している原理と同じ。

## 2. 体験設計原則

| 優先度 | 原則 |
|---|---|
| 最優先 | **TikTokシェア性** > 完成度の高さ |
| 高 | **モバイル前提**(縦画面・親指操作・スクショ前提) |
| 高 | **軽量**(ロード速度・データ量) |
| 中 | **3分以内**で診断完了(質問数12〜15) |
| 中 | **結果としての楽曲提示が主役**(タイプ説明は脇役) |

---

## 3. ビジュアル指示(暫定)

- 結果画面は**スクショされる前提**でデザインする(SNSのトップ画として成立する画づくり)
- 各タイプに固有のカラー / モチーフ / イラストを割り当てる(差別化と"自分のタイプを推したくなる"心理)
- ブランド「ラブソング診断16」のトーン(明るいポップ・白基調・タイプ色を主役・丸み)に準拠

## 4. 動作モード(Claude/CEO)

[CEO/AIエージェントモード](#) (詳細はメモリー側 `feedback_ceo_agent_mode.md` 参照):

### 自走範囲(勝手に進める)
- 質問文・解説文・コード・選曲案などの**生成物の試作**
- 類義語・語順・配色等の**低リスク微調整**
- フェーズ終了時の**次フェーズ自動着手**
- マイルストーンと残タスクの**可視化**

### 必ず確認する範囲
- ブランドトーン/コンセプト変更
- ファウンダーの意向が必要な仕様変更
- 外部公開・対外コミット
- 大量の労力が無駄になりかねない方向転換
- ファイル/コードの大規模破壊的変更
