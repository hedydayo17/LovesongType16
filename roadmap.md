## Phase 4:楽曲レコメンドエンジン構築 [一部クライアント承認必須]

**AI 自動レコメンドアーキテクチャ**で構築。手選定はせず、AIで一気にプール生成。

### 3層構造
1. **Layer 1: 楽曲ソース** = Spotify / Apple Music API から 5,000〜10,000曲取得
2. **Layer 2: AI タグ付け** = LLM が各曲に16タイプ適合度スコア付与
3. **Layer 3: 軽い人間監修** = ファウンダーが「明らかな外れ」だけ除去

### 実装ステップ
- [x] **4.1 ストリーミングAPI選定** Spotify Web API 単体で確定(2026-05-20)
- [ ] **4.2 ソース楽曲収集スクリプト**(8ジャンル × ジャンル別の母集団取得)
- [ ] **4.3 LLM タグ付けスクリプト**(タイプ適合度スコア付与)
- [ ] **4.4 監修 UI**(ファウンダーが NG ボタン押せる画面、軽くて良い)
- [ ] **4.5 結果生成ロジック**(タイプ × ジャンル × スコア閾値 → ランダム10曲)
- [ ] **4.6 再シャッフル機構**(「別の10曲」ボタン)
- [ ] **4.7 新譜の自動追加バッチ**(週次/月次)

### コスト見積
- 初期構築:LLM タグ付け 約 $50 + API 利用料(無料枠 + 範囲内)
- ランニング:月数ドル

### リスク
- **ジャニーズ系の楽曲カバー不完全**(2023年解禁・古い曲未配信)。新譜中心 or YouTube Music API 併用で対応。
- **API 利用規約**:Spotify/Apple Music の利用規約で「診断結果としてのレコメンド表示」が許諾範囲か要確認。

### 確認事項
- [ ] ストリーミングAPI選定(Spotify / Apple Music / YouTube Music / 複数併用)
- [ ] 結果ページでの楽曲表示形式(プレビュー再生? ジャケット表示? リンクのみ?)
- [ ] プレイリスト連携先(マイページ作成 or 単純リンク)
- [ ] 各サービスの利用規約確認
